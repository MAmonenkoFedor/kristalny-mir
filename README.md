# Кристальный Мир — корпоративный сайт

Современный сайт для **ООО «Кристальный Мир Трейд»** (Минск, Беларусь).
Компания оказывает три B2B-услуги: **коммерческий клининг**, **независимая инвентаризация ТМЦ** и **аутсорсинг персонала**.
Стек: **Next.js 15** (фронтенд) + **NestJS** (бэкенд для заявок и админки) + **Prisma/SQLite**.

### Страницы

- `/` — лендинг (Hero, услуги, процесс, преимущества, форма заявки)
- `/services` — услуги: 3 направления, FAQ, CTA
- `/about` — о компании, принципы, цифры, реквизиты
- `/gallery` — галерея проектов
- `/blog` и `/blog/[slug]` — блог и статьи
- `/contact` — контакты и форма заявки
- `/admin` — админка заявок (вход по токену)

```
kristalny-mir/
├─ frontend/   # Next.js 15 (App Router, TypeScript, Tailwind) — лендинг + /admin
└─ backend/    # NestJS — API заявок, хранение в SQLite (Prisma)
```

## Быстрый старт

Нужен Node.js 18.18+ (лучше 20+).

### 1. Бэкенд (NestJS)

```bash
cd backend
cp .env.example .env          # задай ADMIN_EMAIL, ADMIN_PASSWORD, JWT_SECRET
npm install
npx prisma migrate dev --name cms    # создаст БД и таблицы (leads, users, контент)
npm run start:dev             # API на http://localhost:4000
```

> При первом запуске бэкенд автоматически создаёт администратора (из `ADMIN_EMAIL`/
> `ADMIN_PASSWORD`) и засевает дефолтный контент сайта.

Основные эндпоинты:

| Метод | Путь                         | Доступ      | Назначение                          |
|-------|------------------------------|-------------|-------------------------------------|
| POST  | `/auth/login`                | публичный   | вход в админку (email + пароль → JWT)|
| POST  | `/leads`                     | публичный   | приём заявки с формы                 |
| GET   | `/leads`, `PATCH /leads/:id` | JWT         | заявки и смена статуса               |
| GET   | `/content/site\|services\|gallery\|posts` | публичный | контент для сайта        |
| PUT/POST/DELETE | `/admin/content/*`  | JWT         | редактирование контента (CMS)        |
| POST  | `/uploads`                   | JWT         | загрузка изображений                 |

Защищённые запросы — заголовок `Authorization: Bearer <JWT>`.

### 2. Фронтенд (Next.js)

```bash
cd frontend
cp .env.example .env.local    # NEXT_PUBLIC_API_URL=http://localhost:4000
npm install
npm run dev                   # сайт на http://localhost:3000
```

- Лендинг: `http://localhost:3000`
- Админка: `http://localhost:3000/admin` (вход по `ADMIN_EMAIL` / `ADMIN_PASSWORD`)

> Сайт берёт контент из API. Если бэкенд недоступен, страницы показывают
> встроенный контент по умолчанию (фолбэк), поэтому ничего не «ломается».

## Админка (CMS)

`/admin` — вход по логину и паролю. Вкладки:

- **Заявки** — список заявок, смена статуса.
- **Сайт** — логотип, название, телефон/почта/адрес, реквизиты, тексты hero,
  цифры, преимущества, текст футера.
- **Услуги** — добавление/редактирование/удаление услуг.
- **Галерея** — карточки проектов с загрузкой изображений.
- **Блог** — статьи с обложкой и блоками текста, черновики/публикация.

Изображения и логотип загружаются файлом — бэкенд сохраняет их в `backend/uploads`
и раздаёт по `/uploads/...`. После сохранения изменения видны на сайте после обновления страницы.

## Уведомления о заявках (email + Telegram)

При новой заявке бэкенд параллельно шлёт уведомление на почту и в Telegram.
Оба канала опциональны: если переменные не заданы — канал просто отключается,
заявка всё равно сохраняется в БД и видна в админке.

**Email (Gmail):**

1. Включите двухфакторную аутентификацию в аккаунте Google.
2. Создайте «Пароль приложения»: https://myaccount.google.com/apppasswords
3. В `backend/.env` укажите `SMTP_USER`, `SMTP_PASS` (этот пароль), `MAIL_TO`.

**Telegram:**

1. Создайте бота у [@BotFather](https://t.me/BotFather) → получите `TELEGRAM_BOT_TOKEN`.
2. Напишите своему боту любое сообщение.
3. Узнайте свой `TELEGRAM_CHAT_ID` (например, через [@userinfobot](https://t.me/userinfobot)).
4. Впишите оба значения в `backend/.env`.

После правок `.env` перезапустите бэкенд. Все переменные — в `backend/.env.example`.

> Уведомления отправляются в фоне и не задерживают ответ формы; ошибки доставки
> логируются, но не ломают приём заявки.

## Дизайн

«Свежий премиум»: светлая воздушная палитра, акцент — свежая аква (бирюза) в OKLCH,
тонированные нейтрали, мягкие тени, плавный моушн на кривых ease-out-quart.
Принципы взяты из anti-slop дизайн-философий (Impeccable, Emil Kowalski):
никаких фиолетовых градиентов, карточек-в-карточках, bounce-анимаций; только
кастомные easing-кривые, чёткая типографическая иерархия и осмысленные состояния.

> Прототип-референс (структура): Figma Make «Cleaning Services Website».
> Стек и визуал подняты с нуля.
